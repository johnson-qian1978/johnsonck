## 2. 系统总体架构

### 2.1 架构设计原则

1. **分层架构**: 清晰的分层结构，便于维护和扩展
2. **前后端分离**: 前端负责展示，后端负责业务逻辑和数据访问
3. **RESTful API**: 标准化的接口设计，支持多终端接入
4. **高内聚低耦合**: 模块间松耦合，模块内高内聚
5. **可扩展性**: 支持水平扩展和垂直扩展
6. **安全性**: 多层次安全防护机制
7. **可维护性**: 代码规范、日志完善、监控到位

### 2.3 系统分层说明

#### 2.3.1 用户层
- **PC 浏览器**: Chrome、Firefox、Edge、Safari 等现代浏览器
- **移动端**: H5 页面、微信小程序（后续扩展）
- **第三方应用**: 通过 REST API 集成的外部系统

#### 2.3.2 接入层
- **Nginx/Apache**: 反向代理、负载均衡、静态资源服务
- **SSL 终止**: HTTPS 加密传输
- **WAF**: Web 应用防火墙（可选）

#### 2.3.3 应用层
- **Controller 层**: 接收 HTTP 请求，参数校验，调用 Service 层
- **Service 层**: 业务逻辑处理，事务管理，权限控制
- **Mapper 层**: 数据访问，MyBatis Plus ORM，缓存管理

#### 2.3.4 数据层
- **Oracle 数据库**: 存储结构化数据（站点、栏目、内容、用户等）
- **Redis 缓存**: 存储热点数据、会话信息、页面缓存
- **文件系统**: 存储上传的文件、图片、模板、静态化页面等

## 5. 接口设计

### 5.1 RESTful API 设计规范

#### 5.1.1 URL 命名规范
- 使用小写字母和连字符：`/api/site-management/sites`
- 资源名使用复数：`/api/sites`, `/api/channels`
- 嵌套资源体现层级：`/api/sites/{siteId}/channels`

#### 5.1.2 HTTP 方法语义
| 方法 | 用途 | 幂等性 |
|------|------|--------|
| GET | 查询资源 | 是 |
| POST | 创建资源 | 否 |
| PUT | 更新资源（全量） | 是 |
| PATCH | 更新资源（部分） | 是 |
| DELETE | 删除资源 | 是 |

#### 5.1.3 统一响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": { },
  "timestamp": 1711267200000,
  "traceId": "abc123"
}
```

### 5.2 核心 API 分类

| 分类 | 路径前缀 | 说明 |
|------|---------|------|
| 认证接口 | `/api/auth` | 登录、登出、刷新 Token |
| 站点管理 | `/api/sites` | 站点 CRUD、复制、转移 |
| 栏目管理 | `/api/channels` | 栏目树、导入导出 |
| 内容管理 | `/api/contents` | 内容 CRUD、审核、发布 |
| 素材管理 | `/api/materials` | 文件上传、管理 |
| 用户管理 | `/api/users` | 用户 CRUD、权限 |
| 系统管理 | `/api/system` | 配置、日志、统计 |

完整 API 文档详见《API 接口文档》。

## 6. 安全设计

### 6.1 认证机制

#### 6.1.1 JWT Token 认证
- Access Token 有效期 2 小时
- Refresh Token 有效期 7 天
- 支持无感刷新

#### 6.1.2 Spring Security 配置
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**").permitAll()
                .anyRequest().authenticated()
            )
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            )
            .addFilterBefore(jwtAuthenticationFilter(), UsernamePasswordAuthenticationFilter.class);
        
        return http.build();
    }
}
```

### 6.2 授权机制

#### 6.2.1 RBAC 模型
```
用户 → 角色 → 权限 (菜单/按钮/API)
          ↓
      数据范围 (全部/站点/栏目/本人)
```

#### 6.2.2 权限注解
```java
@PreAuthorize("hasAuthority('admin:list')")
@GetMapping
public Result<List<User>> getUsers() { }
```

### 6.3 数据安全

- 敏感数据 AES 加密存储
- HTTPS 传输加密
- SQL 注入防护（MyBatis 参数化查询）
- XSS 防护（HTML 过滤）
- CSRF Token 验证

### 6.4 审计日志

```java
@Aspect
@Component
public class AuditLogAspect {
    
    @Around("@annotation(auditLog)")
    public Object logAudit(ProceedingJoinPoint pjp, AuditLog auditLog) throws Throwable {
        long startTime = System.currentTimeMillis();
        Object result = pjp.proceed();
        long executionTime = System.currentTimeMillis() - startTime;
        
        // 记录操作日志
        auditLogService.saveLog(auditLog, executionTime);
        
        return result;
    }
}
```

## 7. 部署架构

### 7.1 单机部署（开发环境）

```
Nginx (80/443) → Spring Boot (8080) → Oracle (1521)
                            ↓
                         Redis (6379)
```

### 7.2 集群部署（生产环境）

```
SLB → Nginx 集群 → Spring Boot 应用集群 → Oracle RAC
                     ↓
                  Redis Sentinel
                     ↓
                  文件存储 (NAS/OSS)
```

### 7.3 容器化部署

```yaml
version: '3.8'
services:
  web:
    image: cjcms-web:latest
    ports:
      - "80:80"
  api:
    image: cjcms-api:latest
    environment:
      - SPRING_DATASOURCE_URL=jdbc:oracle:thin:@oracle:1521/ORCL
      - SPRING_REDIS_HOST=redis
  oracle:
    image: oracle/database:19.3.0-ee
  redis:
    image: redis:6-alpine
```

### 7.4 高可用方案

- **应用层**: 多节点 + 负载均衡
- **数据库层**: Oracle RAC/Data Guard
- **缓存层**: Redis Sentinel/Cluster
- **备份**: 每日增量，每周全量
