## 1. 引言

### 1.1 编写目的

本文档依据《GB/T 8567-2006 计算机软件文档编制规范》的要求，详细描述长江 CMS（Content Management System，内容管理系统）V2.0 版本的功能需求、性能需求、接口需求、安全需求和运行环境要求。

本文档的编写旨在：
1. 明确系统的需求范围和功能边界
2. 为系统设计、开发、测试提供依据
3. 作为项目验收的标准文档
4. 为用户培训和维护提供参考

**预期读者：**
- 项目管理人员：了解项目范围和目标
- 系统设计人员：进行系统架构和详细设计
- 软件开发人员：根据需求进行编码实现
- 测试人员：编写测试计划和测试用例
- 用户代表：确认需求是否符合业务期望
- 质量保证人员：进行质量评审和验收

### 1.2 项目背景

**项目名称**: 长江 CMS 内容管理系统  
**项目提出者**: 长江数据公司  
**项目开发者**: 长江数据公司研发团队  
**用户单位**: 政府企事业单位、社会团体、媒体机构等需要内容管理的组织  

**项目背景:**
随着信息化建设的深入推进，各类组织机构对网站内容管理的需求日益增长。当前市场上的 CMS 系统要么功能过于简单，无法满足复杂的内容管理需求；要么过于复杂，学习和使用成本高昂。因此，开发一套具有自主知识产权、功能完善、易用性强、安全可靠的内容管理系统具有重要的现实意义。

长江 CMS 系统借鉴了业界领先的 SSCMS（SiteServer CMS）v7 版本的先进设计理念，结合国内用户的实际使用习惯，旨在打造一款适合中国国情的企业级内容管理系统。

### 1.3 项目目标

#### 1.3.1 长期目标

构建一套功能完善、性能优越、安全可靠、易于使用的企业级内容管理系统，支持多站点集群管理、灵活的内容模型定制、细粒度的权限控制、完善的静态化生成能力，满足各类组织机构的网站内容管理需求。

#### 1.3.2 阶段目标

**V2.0 版本目标:**
1. 实现可视化的内容管理平台，让非技术人员也能轻松使用
2. 提供灵活的内容模型定义，支持自定义数据结构
3. 建立完善的 RBAC 权限管理体系，确保数据安全
4. 支持 Oracle 数据库，预留国产数据库（达梦、GBase）迁移能力
5. 提供 RESTful API，支持第三方系统集成
6. 采用前后端分离架构，便于系统扩展和维护
7. 符合等保三级和密评三级安全要求
8. 支持完整的站群管理、栏目管理、内容管理、素材管理、模板管理、生成管理等功能

## 2. 总体描述

### 2.1 产品视角

长江 CMS 系统是一款独立的企业级内容管理软件产品，采用 B/S（Browser/Server，浏览器/服务器）架构，支持多站点集群管理。系统定位于中大型企事业单位、政府机关、社会团体的网站内容管理需求。

系统在整体 IT 架构中的位置：
- **上层**: 面向最终用户提供网站内容展示服务
- **中层**: 作为内容管理中台，向上支撑多个网站门户，向下对接数据存储
- **下层**: 依赖数据库系统（Oracle）、文件存储系统、缓存系统（Redis）等基础设施

系统与外部系统的关系：
- **与门户网站**: 通过静态化页面或 API 接口提供内容
- **与第三方系统**: 通过 REST API 进行数据交换和集成
- **与社交媒体**: 支持与微信公众号等社交平台集成
- **与认证系统**: 支持对接统一身份认证系统（可选）

## 3. 具体需求

本章详细描述系统的功能需求、数据需求、性能需求、接口需求和安全需求。

### 3.1 功能需求

#### 3.1.1 站群管理功能需求

**FR-SSM-001: 创建新站点**
- **优先级**: P0（最高）
- **功能描述**: 支持通过多种方式创建新站点
- **输入**: 站点名称、站点类型、模板选择
- **处理**: 
  1. 验证站点名称唯一性
  2. 根据选择的创建方式初始化站点
  3. 创建站点相关数据表和目录结构
  4. 记录创建日志
- **输出**: 创建成功的站点
- **业务规则**:
  - 站点名称在系统内必须唯一
  - 站点编码只能包含字母、数字和下划线

### 4.1 站群管理模块

#### 4.1.1 功能子项详细需求

**FR-SSM-001: 创建新站点**

- **输入**: 站点名称、站点编码、站点类型、模板选择
- **处理过程**:
  1. 验证站点名称和编码的唯一性
  2. 根据选择的创建方式执行不同逻辑:
     - 本地模板创建：从已有模板复制站点结构和内容
     - 在线模板创建：从官网模板库下载并应用模板
     - 空站点创建：仅创建基础结构，不包含任何内容
  3. 创建站点相关的数据表和目录结构
  4. 初始化站点配置参数
  5. 创建默认栏目（根栏目）
  6. 记录创建日志
- **输出**: 创建成功的站点，返回站点 ID 和访问地址
- **业务规则**:
  - 站点名称在系统内必须唯一
  - 站点编码只能包含字母、数字和下划线，长度不超过 50 字符
  - 创建站点时自动创建默认栏目和默认内容模型
  - 只有超级管理员和站点管理员角色可以创建站点
- **异常处理**:
  - 站点名称或编码已存在：提示错误，要求重新输入
  - 模板文件损坏：提示模板无效，终止创建
  - 磁盘空间不足：提示空间不足，终止创建

**FR-SSM-002: 站群统一管理**
- **功能描述**: 以树形结构统一管理所有站点，支持无限层级
- **输入**: 站点操作指令（新增、修改、删除、转移）
- **处理过程**:
  1. 以树形结构展示所有站点及其层级关系
  2. 支持展开/折叠站点树
  3. 支持站点的增删改查操作
  4. 支持拖拽调整站点层级关系
  5. 支持批量修改站点配置
  6. 显示站点基本信息（名称、状态、内容数量等）
- **输出**: 更新后的站点树和站点信息
- **业务规则**:
  - 支持无限级父子站点关系
  - 删除父站点时需确认是否同时删除子站点
  - 站点转移不能形成循环引用
  - 只能转移有权限管理的站点
- **界面要求**:
  - 树形控件清晰展示层级关系
  - 支持右键菜单操作
  - 支持搜索和过滤站点

**FR-SSM-003: 站点访问地址管理**
- **功能描述**: 配置站点的访问地址和部署方式
- **输入**: 部署方式、访问 URL、上传文件 URL
- **处理过程**:
  1. 选择站点部署方式（本地部署/远程部署）
  2. 配置站点主访问 URL
  3. 配置备用访问 URL（可选）
  4. 配置上传文件访问 URL
  5. 验证 URL 格式和可达性
  6. 保存配置并更新站点信息
- **输出**: 站点访问配置信息
- **业务规则**:
  - 每个站点至少配置一个访问地址
  - 支持主域名和多个备用域名
  - URL 必须以 http://或 https://开头
  - 上传文件 URL 可以与站点 URL 不同（用于 CDN 场景）
- **验证规则**:
  - URL 格式必须符合 RFC 3986 标准
  - HTTPS 地址需要验证 SSL 证书有效性

**FR-SSM-004: 站点内容表管理**
- **功能描述**: 查看和管理各站点的内容表
- **输入**: 站点 ID
- **处理过程**:
  1. 查询站点所有内容表
  2. 显示表名、表注释、字段数量、记录总数
  3. 支持查看表结构详情
  4. 支持预览表数据（脱敏显示）
  5. 统计各表占用空间
- **输出**: 内容表列表及详细信息
- **业务规则**:
  - 只能查看，不能直接修改表结构
  - 敏感字段数据（如密码、手机号）需要脱敏显示
  - 大数据量表查询需要限制返回条数（最多 100 条）
- **性能要求**:
  - 表结构查询响应时间≤2 秒
  - 数据预览响应时间≤5 秒

**FR-SSM-005: 站点模板管理**
- **功能描述**: 管理站点模板文件，支持导入、导出、压缩、解压
- **输入**: 模板文件、操作指令
- **处理过程**:
  1. 导入模板：上传 ZIP 格式的模板压缩包，解压到模板目录
  2. 压缩模板：将模板目录打包成 ZIP 文件供下载
  3. 解压模板：解压已上传的模板压缩包
  4. 创建模板：在线创建新的模板文件
  5. 删除模板：删除不需要的模板（需确认）
  6. 列出所有模板文件及目录结构
- **输出**: 模板文件列表和操作结果
- **业务规则**:
  - 模板压缩包必须是 ZIP 格式
  - 删除模板前需确认是否被栏目或内容使用
  - 模板文件大小不能超过 1GB
  - 模板文件命名必须符合规范（只能包含字母、数字、下划线、点）
- **安全要求**:
  - 上传的压缩包需要进行病毒扫描
  - 禁止上传包含可执行文件的模板包

**FR-SSM-006: 在线站点模板库**

- **处理过程**:
  1. 连接官网模板库 API
  2. 根据条件搜索和筛选模板
  3. 显示模板缩略图和基本信息
  4. 在线预览模板效果（多页面预览）
  5. 下载并应用选中的模板
  6. 记录模板使用历史
- **输出**: 下载的模板和创建的站点
- **业务规则**:
  - 需要网络连接才能访问在线模板库
  - 部分模板可能需要付费购买
  - 免费模板可以直接下载使用
  - 付费模板需要验证授权状态
- **界面要求**:
  - 网格布局展示模板缩略图
  - 支持模板详情弹窗预览

#### 4.2.1 功能子项详细需求

**FR-CHN-001: 添加栏目**

- **处理过程**:
  1. 单个添加：填写栏目详细信息，保存到数据库
  2. 批量添加：输入多行栏目名称（每行一个），批量创建
  3. 自动生成栏目索引（如果未填写）
  4. 设置默认栏目配置
  5. 记录操作日志
- **输出**: 创建成功的栏目
- **业务规则**:
  - 栏目名称在同一父栏目下必须唯一
  - 栏目索引只能包含字母、数字、下划线和连字符
  - 支持无限级栏目层级
  - 栏目索引在网站范围内必须唯一
- **验证规则**:
  - 栏目名称长度：2-100 字符
  - 栏目索引长度：2-50 字符

**FR-CHN-002: 导出栏目**
- **功能描述**: 将栏目及其内容导出为压缩包
- **输入**: 栏目 ID、导出选项
- **处理过程**:
  1. 选择要导出的栏目
  2. 递归获取所有子栏目
  3. 导出栏目配置信息（JSON 格式）
  4. 导出栏目下的所有内容
  5. 打包内容关联的图片、视频、附件等文件
  6. 生成 ZIP 压缩包并提供下载
- **输出**: 栏目导出压缩包
- **业务规则**:
  - 导出内容包括栏目配置、内容数据、关联文件
  - 支持选择是否导出子栏目
  - 支持选择是否导出关联文件
  - 大文件导出采用异步处理方式
- **性能要求**:
  - 小数据量（<100MB）导出时间≤30 秒
  - 大数据量导出提供进度条显示

**FR-CHN-003: 导入栏目**
- **功能描述**: 从压缩包导入栏目及其内容
- **输入**: 栏目压缩包文件
- **处理过程**:
  1. 上传栏目压缩包
  2. 验证压缩包格式和内容
  3. 解析栏目配置信息
  4. 创建栏目结构
  5. 导入内容数据
  6. 解压并保存关联文件
  7. 记录导入日志
- **输出**: 导入成功的栏目和内容
- **业务规则**:
  - 压缩包必须是系统导出的格式
  - 导入时自动处理栏目索引冲突（自动追加后缀）
  - 支持选择导入到指定父栏目下
  - 大文件导入采用异步处理方式
- **异常处理**:
  - 压缩包格式错误：提示文件格式不正确
  - 导入失败：回滚已导入的数据，提示错误原因

**FR-CHN-004: 栏目编辑**
- **功能描述**: 编辑栏目的各项属性和配置
- **输入**: 栏目 ID、更新的栏目信息
- **处理过程**:
  1. 加载栏目详细信息
  2. 显示栏目配置表单（多标签页）
  3. 基本设置：名称、索引、图标等
  4. 模板设置：栏目模板、内容模板选择
  5. 路径设置：访问路径规则配置
  6. SEO 设置：标题、关键词、描述
  7. 权限设置：可见范围、投稿权限
  8. 保存更新并清除相关缓存
- **输出**: 更新后的栏目信息
- **业务规则**:
  - 栏目索引修改后，原链接自动跳转到新链接
  - 模板变更不影响已有内容
  - 权限变更立即生效

  - 分标签页组织配置项，避免表单过长

**FR-CHN-005: 查找栏目**

- **处理过程**:
  1. 接收搜索条件
  2. 模糊匹配栏目名称
  3. 精确匹配栏目 ID 或索引
  4. 高亮显示匹配的栏目
  5. 自动展开栏目树定位到目标栏目
- **输出**: 匹配的栏目列表
- **业务规则**:
  - 支持按名称模糊搜索
  - 支持按 ID 或索引精确搜索
  - 搜索结果按匹配度排序

  - 搜索响应时间≤1 秒

  6. 记录生成日志

- **业务规则**:
  - 首页模板名为 index.html
  - 生成后自动清除缓存
- **性能要求**:
  - 单页面生成时间≤2 秒

**FR-GEN-002: 生成栏目页**
- **功能描述**: 解析栏目模板并生成静态页面
- **输入**: 栏目 ID
- **处理过程**:
  1. 加载栏目模板
  2. 获取栏目数据
  3. 获取栏目下的内容列表
  4. 解析 STL 标签
  5. 生成分页（如需要）
  6. 生成 HTML 文件
  7. 上传到服务器
- **输出**: 生成的栏目页文件
- **业务规则**:
  - 支持分页生成
  - 分页数量可配置
- **性能要求**:
  - 单页面生成时间≤2 秒

**FR-GEN-003: 生成内容页**
- **功能描述**: 解析内容模板并生成静态页面
- **输入**: 内容 ID
- **处理过程**:
  1. 加载内容模板
  2. 获取内容数据
  3. 解析 STL 标签
  4. 生成 HTML 文件
  5. 上传到服务器
- **输出**: 生成的内容页文件
- **业务规则**:
  - 内容页通常不分页
  - 支持长内容自动分页
- **性能要求**:
  - 单页面生成时间≤1 秒

**FR-GEN-004: 生成文件页**
- **功能描述**: 解析文件模板并生成文件详情页
- **输入**: 文件 ID
- **处理过程**:
  1. 加载文件模板
  2. 获取文件数据
  3. 解析 STL 标签
  4. 生成 HTML 文件
- **输出**: 生成的文件页
- **业务规则**:
  - 文件页用于展示 downloadable 文件
  - 包含下载链接和统计

**FR-GEN-005: 生成专题**
- **功能描述**: 解析专题并生成专题页面
- **输入**: 专题 ID
- **处理过程**:
  1. 加载专题配置
  2. 解析专题模板
  3. 替换动态内容
  4. 生成 HTML 文件
  5. 复制关联资源
- **输出**: 生成的专题页面
- **业务规则**:
  - 专题可以包含多个页面
  - 资源文件一并复制

**FR-GEN-006: 一键生成**
- **功能描述**: 一键生成站点所有静态页面
- **输入**: 站点 ID
- **处理过程**:
  1. 获取站点所有栏目
  2. 生成首页
  3. 遍历栏目生成栏目页
  4. 遍历内容生成内容页
  5. 生成文件页
  6. 生成专题页
  7. 显示生成进度
  8. 生成完成后汇总报告
- **输出**: 生成结果报告
- **业务规则**:
  - 支持断点续传
  - 失败的任务可以重试
  - 后台异步执行
- **性能要求**:
  - 千页面级生成时间≤30 分钟
- **界面要求**:
  - 实时显示生成进度
  - 显示成功/失败数量
  - 支持取消生成

**FR-GEN-007: 生成进度查看**
- **功能描述**: 查看页面生成进度和状态
- **输入**: 生成任务 ID
- **处理过程**:
  1. 查询生成任务状态
  2. 显示进度百分比
  3. 显示当前正在生成的页面
  4. 显示成功/失败统计
  5. 支持取消生成任务
- **输出**: 生成进度信息
- **业务规则**:
  - 实时更新进度（每秒刷新）
  - 失败的任务显示错误原因
  - 可以查看详细日志
- **界面要求**:
  - 进度条直观显示
  - 支持暂停和恢复

## 5. 数据需求

### 5.1 数据模型

系统采用关系型数据库存储数据，主要数据实体包括：

1. **站点（Site）**: 描述网站的基本信息
2. **栏目（Channel）**: 描述内容的分类结构
3. **内容（Content）**: 描述具体的文章、新闻等内容
4. **内容模型（Model）**: 定义内容的字段结构
5. **模板（Template）**: 定义页面的展示样式
6. **管理员（Administrator）**: 后台管理系统用户
7. **角色（Role）**: 权限分组
8. **菜单（Menu）**: 系统功能菜单
9. **用户（User）**: 前台注册用户
10. **素材（Material）**: 图片、视频、音频、文件等资源
11. **标签（Tag）**: 内容标签
12. **日志（Log）**: 系统操作日志和错误日志

### 5.2 数据量预估

| 数据类型 | 初始数据量 | 年增长量 | 保留策略 |
|---------|-----------|---------|---------|
| 站点数据 | ≤ 100 条 | ≤ 50 条 | 永久保留 |
| 栏目数据 | ≤ 1000 条 | ≤ 500 条 | 永久保留 |
| 内容数据 | ≤ 10 万条 | ≤ 20 万条 | 永久保留 |
| 用户数据 | ≤ 1 万条 | ≤ 5 万条 | 永久保留 |
| 素材数据 | ≤ 10GB | ≤ 50GB | 永久保留 |
| 操作日志 | ≤ 100 万条 | ≤ 500 万条 | 保留 6 个月 |
| 错误日志 | ≤ 10 万条 | ≤ 50 万条 | 保留 3 个月 |

### 5.3 数据采集

- 用户手动录入：内容创建、用户注册等
- 系统自动生成：日志、统计数据等
- 批量导入：Excel、Word、压缩包等
- API 接口同步：第三方系统数据

### 5.4 数据管理

- **数据备份**: 每日增量备份，每周全量备份
- **数据恢复**: 支持按时间点恢复
- **数据清理**: 定期清理过期日志和临时数据
- **数据归档**: 历史数据定期归档存储

## 8. 安全需求

### 8.1 身份认证

#### 8.1.1 用户名密码认证
- 用户名唯一性校验
- 密码强度要求（至少 8 位，包含字母和数字）
- 密码加密存储（BCrypt 算法）
- 密码定期更换提醒（可配置）

#### 8.1.2 验证码
- 登录失败 3 次后启用图形验证码
- 验证码有效期 5 分钟
- 验证码自动刷新机制

#### 8.1.3 会话管理
- 基于 JWT 的无状态认证
- Token 有效期 2 小时（可配置）
- 支持 Token 刷新
- 支持强制下线

#### 8.1.4 登录保护
- 登录失败锁定（可配置失败次数和锁定时间）
- 异地登录提醒
- 登录历史记录

### 8.2 访问控制

#### 8.2.1 RBAC 权限模型
- 基于角色的访问控制
- 角色 - 权限 - 用户三层模型
- 支持角色继承

#### 8.2.2 菜单权限
- 目录、菜单、按钮、API 四级权限
- 动态菜单加载
- 权限变更实时生效

#### 8.2.3 数据权限
- 全部数据权限
- 站点级数据权限
- 栏目级数据权限
- 本人数据权限

#### 8.2.4 操作权限
- 查看、新增、修改、删除
- 导入、导出
- 审核、发布
- 权限粒度到按钮级别

### 8.3 数据安全

#### 8.3.1 数据加密
- 密码加密：BCrypt
- 敏感数据加密：AES-256（手机号、邮箱等）
- 传输加密：HTTPS/TLS 1.2+

#### 8.3.2 数据脱敏
- 前台展示脱敏
- 日志记录脱敏
- 导出数据脱敏

#### 8.3.3 数据完整性
- 数据库约束（主键、外键、唯一性）
- 数据校验（类型、长度、格式）
- 事务保证数据一致性

### 8.4 输入验证

#### 8.4.1 SQL 注入防护
- 参数化查询
- ORM 框架防止拼接 SQL

#### 8.4.2 XSS 防护
- 输入内容 HTML 过滤

- CSRF Token 验证

- 文件大小限制

- 用户登录日志
