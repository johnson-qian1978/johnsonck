## 1. 引言

### 1.1 编写目的

本文档描述长江 CMS 系统的数据库设计方案，包括表结构、字段定义、索引设计、关系约束等，为数据库实施和维护提供依据。

**预期读者：**
- 数据库管理员（DBA）
- 后端开发人员
- 系统架构师

### 1.2 背景

**项目名称**: 长江 CMS 内容管理系统  
**数据库类型**: Oracle 11g/12c/19c  
**设计原则**: 
- 符合第三范式（3NF）
- 预留国产化数据库迁移能力
- 高性能、高可用、易维护

## 3. 数据库命名规范

### 3.1 表命名规范

**格式**: `CJ_{模块}_{实体}`

**示例**:
- `CJ_SYS_SITE` - 系统模块 - 站点表
- `CJ_CMS_CHANNEL` - CMS 模块 - 栏目表
- `CJ_ADM_ADMINISTRATOR` - 管理员模块 - 管理员表

**规则**:
- 表名长度不超过 30 个字符（Oracle 限制）
- 使用大写字母和下划线
- 避免使用 Oracle 保留字

### 3.2 字段命名规范

**格式**: `{实体缩写}_{属性}`

**示例**:
- `SITE_ID` - 站点 ID
- `SITE_NAME` - 站点名称
- `CREATE_TIME` - 创建时间

**规则**:
- 字段名长度不超过 30 个字符
- 使用大写字母和下划线
- 主键统一命名为 `{表名缩写}_ID`
- 外键命名为 `FK_{表名}_{关联表名}`

### 3.3 索引命名规范

**格式**: 
- 主键索引：`PK_{表名}`
- 唯一索引：`UK_{表名}_{字段名}`
- 普通索引：`IDX_{表名}_{字段名}`
- 外键索引：`FK_{表名}_{字段名}`

### 3.4 序列命名规范

**格式**: `SEQ_{表名}`

**示例**:
- `SEQ_CJ_SYS_SITE` - 站点表序列
- `SEQ_CJ_CMS_CHANNEL` - 栏目表序列

### 3.5 视图命名规范

**格式**: `V_{表名}_{用途}`

**示例**:
- `V_CJ_CMS_CONTENT_LIST` - 内容列表视图

## 4. 概念结构设计

### 4.1 E-R 图

```
┌─────────────┐       ┌──────────────┐       ┌─────────────┐
│   站点表    │1     n│   栏目表     │1     n│  内容表     │
│  CJ_SYS_SITE│───────│CJ_CMS_CHANNEL│───────│CJ_CMS_CONTENT│
└─────────────┘       └──────────────┘       └─────────────┘
      │1                    │1                    │
      │                     │n                    │n
      │                     │                     │
      │              ┌──────┴──────┐              │
      │              │             │              │
      │         ┌────▼────┐  ┌────▼────┐         │
      │         │内容模型表│  │模板表   │         │
      │         │CJ_CMS_  │  │CJ_CMS_  │         │
      │         │MODEL    │  │TEMPLATE │         │
      │         └─────────┘  └─────────┘         │
      │                                          │
      │         ┌──────────────┐                │
      │         │  管理员表    │                │
      └────────▶│CJ_ADM_       │◀───────────────┘
                │ADMINISTRATOR │
                └──────────────┘
```

### 4.2 实体关系说明

1. **站点与栏目**: 一对多关系，一个站点包含多个栏目
2. **栏目与内容**: 一对多关系，一个栏目包含多条内容
3. **栏目与栏目**: 自引用一对多关系，支持树形结构
4. **内容与内容模型**: 多对一关系，多条内容使用同一个内容模型
5. **站点与模板**: 一对多关系，一个站点包含多个模板
6. **管理员与站点**: 多对多关系，通过权限表关联

#### 5.1.6 内容扩展表 (动态创建)

**命名规则**: `CJ_CMS_CONTENT_{MODEL_CODE}`

**示例**: 如果模型编码为 `NEWS`，则扩展表名为 `CJ_CMS_CONTENT_NEWS`

**基础结构**:
```sql
CREATE TABLE CJ_CMS_CONTENT_NEWS (
    CONTENT_ID VARCHAR2(40) PRIMARY KEY,
    -- 以下为动态字段，根据模型定义生成
    NEWS_CONTENT CLOB,
    NEWS_DATE DATE,
    NEWS_CATEGORY VARCHAR2(50),
    -- ...
    FOREIGN KEY (CONTENT_ID) REFERENCES CJ_CMS_CONTENT(CONTENT_ID)
);
```

## 8. 数据库维护设计

### 8.1 备份策略

**备份计划**:

- **每周全量备份**: 每周日凌晨 1:00

**备份脚本示例**:
```bash
#!/bin/bash
# 全量备份脚本
RMAN TARGET / << EOF
RUN {
    ALLOCATE CHANNEL c1 TYPE DISK;
    BACKUP FULL FORMAT '/backup/cjcms_full_%d_%T_%U.bak' DATABASE;
    BACKUP FORMAT '/backup/cjcms_arch_%d_%T_%U.bak' ARCHIVELOG ALL;
    RELEASE CHANNEL c1;
}
EOF
```

### 8.2 性能监控

**监控指标**:
- 表空间使用率
- 慢查询日志
- 锁等待情况
- CPU/内存使用率
- I/O 性能

**监控 SQL 示例**:
```sql
-- 查询表空间使用率
SELECT tablespace_name, 
       ROUND(used_space * 8192 / 1024 / 1024, 2) used_mb,
       ROUND(tablespace_size * 8192 / 1024 / 1024, 2) total_mb,
       ROUND(used_percent, 2) used_percent
FROM dba_tablespace_usage_metrics;

-- 查询慢查询
SELECT sql_id, elapsed_time, executions, sql_text
FROM v$sql
WHERE elapsed_time > 10000000  -- 大于 10 秒
ORDER BY elapsed_time DESC;
```

### 8.3 数据清理

**定期清理策略**:
- 日志表保留最近 6 个月数据
- 临时表每日清理
- 过期会话定期清理

**清理脚本示例**:
```sql
-- 清理 6 个月前的操作日志
DELETE FROM CJ_SYS_OPERATION_LOG 
WHERE CREATE_TIME < ADD_MONTHS(SYSDATE, -6);

-- 清理 30 天前的错误日志
DELETE FROM CJ_SYS_ERROR_LOG 
WHERE CREATE_TIME < ADD_MONTHS(SYSDATE, -1);
```
