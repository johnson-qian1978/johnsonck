# 菜单管理系统 - 需求规格说明书

## 📋 项目概述

为武汉医疗互助系统开发一个**菜单管理**功能，用于管理系统中的菜单项和按钮权限。

---

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| **MENU_ID** | VARCHAR2(40) | 主键，UUID | `2f32c4c0-e631-4b8e-ac08-dbc8bc8e7014` |

## 🔗 表关系

```
AUTH_MENU (1) ────< AUTH_MENU_BUTTON (>1)
                        │
                        │ (引用按钮名称，非外键)
                        ▼
                  AUTH_BUTTON (字典表)
```

- **菜单与按钮**: 一对多关系，一个菜单可以有多个按钮
- **按钮定义**: AUTH_BUTTON 是字典表，定义系统中所有可用的按钮类型
- **关联方式**: AUTH_MENU_BUTTON 通过 BUTTON_NAME 关联到 AUTH_BUTTON，不是通过 ID

## 📝 功能需求

### 一、菜单管理

#### 1. 菜单列表展示
- [ ] 树形结构展示菜单（支持多级菜单）
- [ ] 显示字段：菜单名称、路径、组件、图标、排序、显示状态
- [ ] 支持展开/折叠树节点
- [ ] 根菜单 PARENT_ID = '0'

#### 2. 新增菜单
- [ ] 选择父菜单（可为根菜单）
- [ ] 填写菜单标题（必填）
- [ ] 填写组件名称（必填）
- [ ] 填写路由路径（可选）
- [ ] 选择图标（可选）
- [ ] 设置排序号（必填，默认 0）
- [ ] 设置显示/隐藏（默认显示）
- [ ] 自动生成 MENU_ID (UUID)

#### 3. 编辑菜单
- [ ] 可修改除 MENU_ID 外的所有字段
- [ ] 不允许修改为自身的子菜单（避免循环引用）

#### 4. 删除菜单
- [ ] 删除前检查是否有子菜单
- [ ] 删除前检查是否有关联按钮
- [ ] 二次确认提示

#### 5. 菜单排序
- [ ] 支持拖拽排序或上下移动
- [ ] 更新 MENU_NO 字段

#### 6. 显示/隐藏切换
- [ ] 快速切换 SHOW_FLAG (0/1)

---

### 二、按钮管理

#### 1. 按钮列表（按菜单分组）
- [ ] 选择菜单后显示该菜单下的所有按钮
- [ ] 显示字段：按钮名称、标识、URL、排序

#### 2. 添加按钮到菜单
- [ ] 从 AUTH_BUTTON 字典表中选择按钮类型
- [ ] 填写按钮标识（BUTTON_SIGN）
- [ ] 填写按钮 URL/动作
- [ ] 设置按钮顺序

#### 3. 移除按钮
- [ ] 从菜单中移除按钮关联
- [ ] 不影响 AUTH_BUTTON 字典表

---

### 三、界面要求

#### 布局参考 BMD 管理界面
- [ ] 左侧导航：系统管理 > 菜单管理
- [ ] 左右分栏布局：
  - **左侧**: 菜单树形列表
  - **右侧**: 选中菜单的详细信息 + 按钮配置

#### 交互细节
- [ ] 点击菜单树节点，右侧显示详情
- [ ] 支持菜单的新增/编辑/删除
- [ ] 支持按钮的添加/移除
- [ ] 实时保存或确认后保存

## ✅ 验收标准

1. **菜单 CRUD**: 可以正常新增、编辑、删除菜单
2. **树形展示**: 正确显示父子关系，支持多级菜单
3. **按钮关联**: 可以为菜单添加/移除按钮
4. **数据一致性**: 删除菜单时检查子菜单和关联按钮
5. **界面美观**: 符合现有系统风格（武汉医疗互助）
